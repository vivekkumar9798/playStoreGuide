Before You Start – Complete Checklist

How to Generate Upload Keystore (Do This Only Once!)

Method 1: Android Studio (Recommended for Beginners)
  1. Open Android Studio → Build → Generate Signed Bundle / APK
  2. Select “Android App Bundle” → Next
  3. Click “Create new…”
  4. Fill: Key store path, passwords, alias: upload, validity: 100 years
  5. Fill certificate info → OK → Finish
Method 2: Terminal Command
keytool -genkey -v -keystore my-upload-key.jks -alias upload -keyalg RSA -keysize 2048 -validity 10000
BACKUP YOUR KEYSTORE EVERYWHERE!
Google Drive + USB + Email + Cloud. If you lose it → You can NEVER update your app again!

1. Log in to Google Play Console

https://play.google.com/console

Use the Google account that paid the $25 developer fee.

2. Create a New App

Click blue “Create app” button → Fill name (≤30 chars), language, type, free/paid → Create app

3. Complete Main Store Listing

  • Title ≤ 30 chars
  • Short description ≤ 80 chars
  • Full description ≤ 4000 chars
  • App icon 512×512 PNG (no transparency)
  • Feature graphic 1024×500
  • Minimum 2 screenshots

4. Add Privacy Policy URL (Mandatory)

Go to Policy → Privacy policy

Free generators:
freeprivacypolicy.com
Firebase Generator

5. Complete App Content Section

All required:
• Data safety (be 100% honest)
• Content rating
• Target audience
• Ads declaration

6. Go to Production Track

Release → Production → Create new release

7. Choose App Signing

RECOMMENDED: “Let Google manage and protect your app signing key” (Safest option)

8. Final build.gradle + Generate & Upload .AAB

Perfect app/build.gradle (2025 Working)
android {
    defaultConfig {
        applicationId "com.yourcompany.yourapp"
        versionCode 35
        versionName "6.0.0"
    }
    signingConfigs {
        release {
            storeFile file("C:/keys/my-upload-key.jks")
            storePassword "yourpassword"
            keyAlias "upload"
            keyPassword "yourpassword"
            v1SigningEnabled true
            v2SigningEnabled true
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled true
            shrinkResources true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
Generate Release AAB:
./gradlew clean
./gradlew bundleRelease

AAB Location: app/build/outputs/bundle/release/app-release.aab

9. Add Release Notes

Write what's new in this version (English + other languages if possible)

10. Review Release & Start Rollout

Congratulations!
Click “Review release” → Fix any red warnings → Click “Start rollout to Production”

Your app is now submitted!

11. Review Timeline

  • First-time apps: 3–7 days (sometimes longer)
  • Future updates: Usually within 24 hours
  • You’ll get email when approved/rejected

Most Common Errors & Fixes

You uploaded a debug-signed bundle → Fix: Run ./gradlew bundleRelease
Version code already used → Fix: Increase versionCode by 1
Missing Privacy Policy / Data Safety → Complete all required sections
